# 11.7 依赖聚簇划分

> 按依赖关系 BFS 聚簇替代文件夹+角色划分，SubAgent 减少 50-60%。

---

## 背景

`cluster-tasks.ts` 是本优化的实现脚本，详见 [第五章 5.7 节](../05-dependency-phase/07-cluster-tasks.md)。

### 文件夹模式的局限

| 问题 | 示例 |
|:---|:---|
| SubAgent 数量多 | 每文件夹每角色一个 SubAgent |
| 跨文件夹依赖割裂 | `useClick.ts` 被 `Button.tsx` + `Dropdown.tsx` 共用但分属不同任务 |
| 文件夹边界不合理 | `utils/` 工具函数被大量组件引用 |

### 聚簇方案的优势

| 优势 | 原理 |
|:---|:---|
| SubAgent 数量少 50-60% | 依赖 BFS 将关联文件聚为一簇 |
| 上下文更完整 | 跨文件夹的依赖关系被保留在一个聚簇内 |
| 命名更准确 | 按目录多数分布投票决定聚簇名 |

## 收益

| 指标 | 文件夹模式 | 聚簇模式 | 改善 |
|:---|:---:|:---:|:---:|
| SubAgent 数量 | ~28 个/项目 | ~12 个/项目 | **-57%** |
| 跨文件上下文 | 缺失 | 完整 | ✅ |
