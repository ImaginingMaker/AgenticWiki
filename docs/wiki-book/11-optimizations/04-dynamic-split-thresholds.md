# 11.4 动态拆分阈值

> 文件夹拆分策略从硬编码 50K/30K/5K 改为按项目总 Token 的百分比计算。

---

## 背景

50K/30K/5K 的硬编码阈值：

| 阈值 | 用途 | 问题 |
|:---|:---|:---|
| 50K (split) | 超过此值 → 文件夹需拆分 | 100K 的项目每个文件夹都"超标"；5M 的项目无文件夹需拆分 |
| 30K (noSplit) | 低于此值 → 不拆分子任务 | 同上 |
| 5K (mergeMin) | 低于此值 → 跨文件夹合并 | 同上 |

## 方案

阈值 = 项目总 Token × 百分比，并 clamp 到安全范围：

| 阈值 | 百分比 | Clamp 范围 |
|:---|:---:|:---:|
| split | 5% | [20K, 150K] |
| noSplit | 2.5% | [10K, 80K] |
| mergeMin | 0.3% | [3K, 15K] |

### 示例对比

| 项目规模 | split (旧) | split (新) |
|:---|---:|---:|
| 100K Token | 50K | **20K** |
| 500K Token | 50K | **25K** |
| 1M Token | 50K | **50K** |
| 3M Token | 50K | **150K** |
